One subtle but important EJS principle here is the difference between:

* `<%= %>` (escaped output)
* `<%- %>` (unescaped output, which is what makes `include()` work properly for inserting HTML)

Why do you think EJS uses `<%- include(...) %>` instead of `<%= include(...) %>` when inserting partials?

Me 
EJS uses `<%- include(...) %>` instead of `<%= include(...) %>` because partials return HTML content that should be rendered directly, not escaped. The `<%- %>` syntax outputs raw HTML, which allows the included file (such as a footer or header) to be properly inserted into the page structure.
If `<%= %>` were used instead, the HTML would be escaped and displayed as plain text instead of being rendered as actual HTML elements. That is why `<%- include(...) %>` is needed when inserting partials.

Coach
Correct—that’s exactly the key distinction.

* **HTML escaping vs raw rendering**
* Why partials must be injected as real DOM structure
* How EJS controls output safety vs layout composition

Coach
Good example—you’re correctly describing a **responsive design breakpoint** and the intent behind it.

A couple of deeper CSS principles are working here:

* `@media (max-width: 768px)` is a **mobile-first or responsive breakpoint rule**, meaning styles inside it override default styles when the screen shrinks.
* `flex-direction: column` only works if `nav ul` is already using `display: flex` somewhere outside this snippet.
* `gap` is a modern Flexbox feature that replaces older spacing hacks like margins between `<li>` elements.

To connect this to real layout thinking: what is your default (non-mobile) layout for `nav ul`, and where do you define it in your CSS?

The default (non-mobile) layout for the navigation is defined in the main `nav ul` rule.

    display: flex;
    justify-content: center;
    gap: 2rem;
    flex-wrap: wrap;
}

This creates a horizontal navigation bar using Flexbox. The items are aligned in a row, centered, and spaced evenly using `gap: 2rem`. This is the standard desktop layout. The mobile version is then handled separately using a media query, which changes the layout to a vertical column.
